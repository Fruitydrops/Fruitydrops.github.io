# .learnings 记录约定

本目录用于记录可复用的问题、错误和经验。

## 书写原则

- 以有用为先，不以格式完整为先。
- 只记录以后回看还能帮助定位和决策的信息。
- 不为了凑模板强行补字段。

## Metadata 规则

- `Metadata` 不是必须写满。
- 只有在确实有帮助时才写元数据。
- 如果某个字段不能解释“为什么和这条记录强相关”，就不要写。

## Related Files 规则

- 只有文件与问题根因、排查路径、修复动作直接相关时才写。
- 不能只是“这次看过这个文件”就写进去。
- 回看时如果不能帮助定位问题，就不写。

## Tags 规则

- 只有在后续检索有价值时才写。
- 泛泛而谈、重复度高、没有筛选价值的标签不要写。

## Pattern-Key 规则

- 只有在这条记录已经形成稳定可复用模式时才写。
- 一次性问题、偶发现象不要强行抽象成 Pattern-Key。

## 建议

- 优先写清根因、现象、排查方法、正确操作。
- 能用一句话说清的，不要展开成一大段。
